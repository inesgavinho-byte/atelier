# DECIMA Workspace — Foundational Architecture

## Vision

**DECIMA Workspace** is not another AI chat.

It is not a dashboard.

It is not a document manager.

It is the operational environment where humans work every day.

While people work, **DECIMA** continuously transforms that activity into organisational intelligence.

The Workspace exists for humans.

DECIMA exists for knowledge.

---

# The Architecture

Human → DECIMA Workspace → Spaces → Projects → Sessions → Documents / Readings / Captures / Images / Emails / Meetings / Decisions → Processing Pipeline → DECIMA → DECIM001

The Workspace is the interface.
DECIMA is the platform.
DECIM001 is the Cognitive Engine.

---

# DECIMA

DECIMA is not an application.

DECIMA is the cognitive platform.

Its responsibility is to:

* preserve memory;
* organise knowledge;
* understand relationships;
* accumulate judgement;
* discover patterns;
* learn continuously.

DECIMA never belongs to one Space.

Every Space feeds DECIMA.

DECIMA becomes progressively more capable over time.

---

# DECIMA Workspace

The Workspace is where people perform their daily work.

Examples:

* writing
* programming
* architecture
* meetings
* emails
* research
* image generation
* document review
* project management

Humans never interact directly with DECIMA.

They interact with DECIMA Workspace.

---

# Spaces

A Space is not a database.

A Space is not a silo.

A Space is an operational context.

Examples: GAVINHO, DECIMA, PAPERS, NUDO, Personal

Every Space has its own: projects, chats, documents, artefacts, decisions, timeline, Minions, AI Sessions.

The Space keeps work organised.

Knowledge never becomes isolated.

---

# Projects

Projects belong to a Space.

Projects become the unit of execution.

---

# Sessions

The chat becomes a Session.

A Session always has: objective, provider, model, Skill, context, outputs.

A Session is never isolated.

It belongs to the Project.

Everything generated becomes part of the Timeline.

---

# Timeline

Every Project has one continuous Timeline.

It records: Sessions, Readings, Documents, Images, Decisions, Commits, Pull Requests, Deployments, Meetings, Emails, Artefacts.

The Timeline becomes the project's memory.

---

# Documents

Every Space contains its own document library.

Each uploaded document follows the same pipeline:

Original File → Permanent Storage → OCR → MarkItDown → Markdown → Chunking → Knowledge Extraction → Indexing → Space Knowledge → DECIMA

Nothing is discarded.

---

# Why MarkItDown?

Large PDFs should never be sent directly to LLMs.

Instead: PDF → Markdown → Knowledge Extraction → Relevant Sections → LLM

Benefits: lower token consumption, cleaner context, searchable content, reusable processing, permanent storage.

---

# Minions

Minions are autonomous continuous workers.

They never replace judgement.

They prepare judgement.

Their mission is to observe, organise and recommend.

Every Minion produces: Signal → Evidence → Interpretation → Recommendation → Approval Required?

---

# First Generation Minions

1. Inbox Minion — processes everything entering DECIMA Workspace
2. Context Minion — maintains each Space continuously updated
3. Decision Minion — identifies decisions requiring approval
4. Project Sentinel — monitors projects, risks, deadlines, dependencies
5. Product Minion — monitors repositories, commits, PRs, architecture
6. Pattern Minion — observes every Space simultaneously, finds invisible relationships

---

# Council

Council coordinates AI.

Humans never choose providers.

They choose intentions.

Examples: Write → Council → Claude | Research → Council → Perplexity | Engineering → Council → DeepSeek

The provider becomes implementation detail.

---

# DECIM001

DECIM001 is not an LLM.

It is the first Cognitive Engine.

Responsibilities: Memory, Knowledge, Relationships, Judgement, Patterns, Learning, Runtime orchestration.

LLMs become interchangeable execution engines.

DECIM001 remains permanent.

---

# Continuous Learning

Every interaction strengthens DECIMA.

Question → Session → Decision → Outcome → Lesson → Knowledge → Pattern → DECIM001

Nothing disappears. Everything teaches the platform.

---

# Core Principle

**Spaces organise work.**

**DECIMA preserves knowledge.**

**DECIM001 learns continuously.**
