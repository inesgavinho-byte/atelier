# AT-0002 — Operating System

## Propósito

Este documento define o funcionamento operacional do ATELIER.

O Operating System descreve como o trabalho intelectual circula pelo sistema, desde a definição de uma iniciativa até à produção de conhecimento e valor.

Não define tecnologia.

Não define interfaces.

Define comportamento.

# 1. Princípios

O funcionamento do ATELIER deverá respeitar sempre:

* Manifesto
* Canon
* Ontologia
* Organização

Nenhum fluxo operacional poderá contrariar estes documentos.

# 2. Estrutura Geral

Todo o trabalho desenvolvido no ATELIER decorre dentro de uma Iniciativa.

Exemplos:

* PAPERS
* DECIMA
* GAVINHO
* NUDO

Cada Iniciativa possui Objetivos.

Cada Objetivo mobiliza Loops.

Cada Loop envolve Agentes.

Os Agentes produzem Artefactos.

Os Artefactos originam Decisões.

As Decisões enriquecem a Memória.

A Memória contribui para o Conhecimento.

O Conhecimento produz Valor.

# 3. O Ciclo Operacional

O ciclo operacional do ATELIER é contínuo.

```
Iniciativa

↓

Objetivo

↓

Loop

↓

Agente

↓

Artefacto

↓

Aprovação

↓

Decisão

↓

Memória

↓

Conhecimento

↓

Valor
```

Sempre que uma nova decisão é tomada, o ciclo reinicia-se.

# 4. Iniciativas

Uma Iniciativa representa uma frente permanente de trabalho.

Cada Iniciativa possui:

* propósito;
* objetivos;
* artefactos;
* memória;
* histórico;
* agentes envolvidos.

As Iniciativas permanecem activas enquanto produzirem valor.

# 5. Objetivos

Os Objetivos representam resultados concretos.

Cada Objetivo deverá possuir:

* descrição;
* responsável;
* estado;
* prioridade;
* prazo, quando aplicável;
* critérios de conclusão.

Um Objetivo pode activar vários Loops.

# 6. Loops

Os Loops representam processos repetíveis.

Exemplos:

* Editorial;
* Investigação;
* Produção;
* Publicação.

Cada Loop possui:

* entrada;
* execução;
* validação;
* saída.

# 7. Agentes

Os Agentes executam trabalho.

Podem:

* investigar;
* produzir;
* organizar;
* sintetizar;
* documentar;
* recomendar.

Não podem alterar conhecimento aprovado sem autorização.

# 8. Artefactos

Todo o trabalho materializado constitui um Artefacto.

Exemplos:

* documentos;
* papers;
* apresentações;
* imagens;
* código;
* prompts;
* modelos.

Cada Artefacto possui:

* autor;
* versão;
* estado;
* histórico;
* relações.

# 9. Estados

Os Artefactos seguem o seguinte ciclo.

```
Draft

↓

Em Desenvolvimento

↓

Em Revisão

↓

Aprovado

↓

Publicado

↓

Arquivado
```

Nem todos os Artefactos percorrem todos os estados.

# 10. Aprovações

As Aprovações representam pontos obrigatórios de julgamento humano.

Uma Aprovação deverá existir sempre que:

* exista impacto público;
* exista impacto estratégico;
* exista alteração constitucional;
* exista alteração de identidade;
* exista publicação.

# 11. Decisões

As Decisões alteram o estado do sistema.

Toda a Decisão deverá preservar:

* contexto;
* responsável;
* fundamentação;
* data;
* impacto.

# 12. Memória

A Memória regista continuamente:

* decisões;
* artefactos;
* versões;
* relações;
* histórico.

A Memória nunca elimina contexto.

# 13. Conhecimento

O Conhecimento resulta da integração da Memória.

O sistema deverá tornar visíveis:

* relações;
* padrões;
* recorrências;
* evolução.

# 14. Valor

O objetivo final do Operating System é produzir Valor.

O Valor poderá assumir natureza:

* intelectual;
* económica;
* estratégica;
* cultural;
* humana.

# 15. Colaboração

Os Offices colaboram continuamente.

Nenhum trabalho deverá depender de um único agente quando puder beneficiar da colaboração entre diferentes funções.

# 16. Autonomia

Os Agentes deverão possuir o maior nível possível de autonomia operacional.

O sistema deverá delegar automaticamente todas as actividades que não exijam julgamento humano.

# 17. Intervenção Humana

O utilizador intervém sempre que:

* existe uma Aprovação;
* existe conflito;
* existe decisão estratégica;
* existe necessidade de orientação.

Todo o restante trabalho deverá ser executado autonomamente.

# 18. Evolução

O Operating System é evolutivo.

Novos Loops.

Novos Offices.

Novos Agentes.

Novas Iniciativas.

Poderão ser adicionados sem alterar os princípios fundamentais definidos na Constituição.
