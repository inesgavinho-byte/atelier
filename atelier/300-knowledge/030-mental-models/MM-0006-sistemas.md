# MM-0006 — Sistemas

## Definição

Um sistema é um conjunto de elementos interligados que produzem um comportamento coletivo.

Para compreender um sistema, não basta analisar as suas partes.

É necessário compreender as relações entre elas.

---

# Significado

Os problemas complexos raramente têm uma causa única.

São consequência da interação entre múltiplos elementos.

Quando uma decisão altera um componente, altera também o comportamento do sistema.

O pensamento sistémico evita soluções locais que degradam o desempenho global.

No ATELIER, praticamente tudo é um sistema:

* uma empresa;
* um projeto;
* uma equipa;
* uma Skill;
* uma Knowledge Library;
* um Agente;
* uma decisão.

---

# Quando usar

Utilizar em decisões de:

* arquitetura;
* estratégia;
* desenvolvimento de produto;
* desenho organizacional;
* liderança;
* processos;
* automação;
* integração de IA.

Sempre que uma alteração possa produzir efeitos indiretos.

---

# Pergunta Principal

**Que sistema estou realmente a alterar?**

---

# Perguntas de Apoio

1. Quais são os elementos deste sistema?
2. Como se relacionam?
3. Onde existem ciclos de feedback?
4. Que comportamento emerge destas relações?
5. O problema está numa parte ou na interação entre partes?
6. Que efeitos secundários esta decisão poderá produzir?
7. Estamos a otimizar uma parte ou o sistema inteiro?
8. O sistema ficará mais resiliente ou mais frágil?

---

# Aplicação

## 1. Definir os limites

Que sistema estamos a analisar?

---

## 2. Identificar os elementos

Pessoas.

Processos.

Tecnologia.

Conhecimento.

Recursos.

---

## 3. Mapear relações

Como influenciam uns aos outros?

---

## 4. Identificar feedback

Que ciclos reforçam ou equilibram o comportamento?

---

## 5. Encontrar pontos de intervenção

Onde uma pequena alteração produz um grande efeito?

---

# Exemplo

Uma equipa entrega projetos com atraso.

A solução aparente é contratar mais pessoas.

O pensamento sistémico mostra que o verdadeiro problema está na forma como as decisões são tomadas.

Ao melhorar o fluxo de decisão, toda a equipa aumenta a capacidade sem crescer.

O sistema mudou.

Não apenas um recurso.

---

# Erros a Evitar

* tratar sintomas em vez da estrutura;
* otimizar departamentos isoladamente;
* ignorar efeitos de longo prazo;
* assumir relações lineares em sistemas complexos;
* procurar culpados em vez de compreender o sistema.

---

# Relação com Princípios

## Reforça

* O conhecimento vive nas relações, não nos documentos.
* Compreender antes de agir.
* Toda a decisão deve produzir aprendizagem.

## Complementa

* Centro de Gravidade.
* Pensamento de Segunda Ordem.
* Alavancagem.

# Fonte

Modelo mental inspirado na Teoria dos Sistemas de Donella Meadows e Peter Senge, adaptado por Inês Gavinho para servir de base ao raciocínio do ATELIER. A sua integração reflete a visão de que o conhecimento, as organizações e os projetos são redes de relações cujo comportamento emerge das interações entre os seus elementos.

---

# Operador Cognitivo

Quando um Agente ativa MM-0006, deve executar automaticamente o seguinte raciocínio:

```text
PROBLEMA

↓

Qual é o sistema em análise?

↓

Quais são os elementos?

↓

Como se relacionam?

↓

Onde existem ciclos de feedback?

↓

Qual é o comportamento emergente?

↓

Onde existe maior alavancagem?

↓

Que intervenção melhora o sistema como um todo?

↓

RECOMENDAÇÃO
```

O objetivo não é otimizar componentes isolados.

É melhorar o comportamento global do sistema.
